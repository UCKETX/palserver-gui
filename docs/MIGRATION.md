# 存檔遷移教學

**繁體中文** | [简体中文](MIGRATION.zh-CN.md)

把既有的 Palworld 世界搬進 palserver GUI 管理的實例。三種來源、同一套原則。

> **開始之前**:所有遷移都要在**伺服器停止**的狀態下進行,而且來源端的遊戲/伺服器也要完全關閉 —— Palworld 執行時會持續寫入存檔,複製到一半的檔案是壞的。

---

## 存檔結構

```
<伺服器目錄>/Pal/Saved/
├─ SaveGames/0/<世界 GUID>/          ← 一個世界一個資料夾
│  ├─ Level.sav                      世界本體(建築、帕魯、地形改變)
│  ├─ LevelMeta.sav
│  ├─ WorldOption.sav                建立世界時的設定快照
│  └─ Players/<PlayerUid>.sav        每位玩家的角色
└─ Config/WindowsServer/
   ├─ PalWorldSettings.ini           世界設定(由 GUI 產生,別手動改)
   └─ GameUserSettings.ini           ★ DedicatedServerName 決定載入哪個世界
```

**最重要的一件事**:伺服器不會自動載入「唯一那個」世界資料夾,而是讀
`GameUserSettings.ini` 裡的 `DedicatedServerName=` 去找對應 GUID 的資料夾。
遷移失敗九成是這裡沒對上 —— 伺服器找不到,就默默開一個全新世界。

在 GUI 裡,**存檔備份**分頁會把每個世界的 GUID 和「啟用中」狀態直接列出來,
按「設為啟用世界」就會幫你改好這一行,不用手動編輯。

---

## 情境 A:從別台專用伺服器搬過來

最單純,格式完全相同。

1. **來源端**:停止舊伺服器。到 `Pal/Saved/SaveGames/0/`,找到你的世界資料夾
   (一串英數字的 GUID)。整個資料夾壓縮成 zip。
2. **目標端**:在 GUI 建立實例後,**先啟動一次再停止** —— Palworld 只有在第一次
   啟動時才會生成 `SaveGames/0/` 和設定檔。
3. GUI → **模組**分頁 → 「瀏覽全部」→ 進到 `Pal/Saved/SaveGames/0/`,
   用「上傳檔案」把 zip 裡的內容放進去(或用「新資料夾」建好 GUID 資料夾後
   上傳裡面的 .sav 檔)。
4. GUI → **存檔備份**分頁 → 找到剛上傳的世界 → 按「**設為啟用世界**」。
5. 啟動伺服器。玩家用原本的角色進來即可。

> **不要直接複製舊的 `GameUserSettings.ini` 過來**。它夾帶了舊主機的
> `PublicIP`、`PublicPort`、RCON 設定,會讓新伺服器列表顯示錯誤或遠端管理失效。
> 只需要改 `DedicatedServerName` 這一行,而這件事 GUI 幫你做。

## 情境 B:從 v1 palserver-GUI 搬過來

v1 把每個伺服器放在自己的資料夾裡,底下的 `Pal/Saved/` 結構和情境 A 完全一樣。

1. 在 v1 介面按「開啟伺服器資料夾」,進到 `Pal/Saved/SaveGames/0/`。
2. 之後照**情境 A 的第 1 步起**操作即可。

如果 v1 的伺服器就在同一台機器上,更省事的做法是**直接收編**:在 GUI 建立實例時,
「既有伺服器路徑」填 v1 那個伺服器目錄(含 `PalServer.exe` 的那一層)。
不用搬任何檔案,世界、模組、設定全部原地接管。

## 情境 C:從本機多人遊戲(四人邀請碼)搬過來

這是唯一會踩到**角色重置 bug** 的情境:本機存檔的 PlayerUid 和專用伺服器的
編碼方式不同,直接搬過去,玩家進來會被要求建立新角色(帕魯和建築還在,
角色本身不見)。

1. 找到本機存檔:
   `%LOCALAPPDATA%\Pal\Saved\SaveGames\<你的 SteamID>\<世界 GUID>\`
2. 照情境 A 上傳到伺服器的 `SaveGames/0/` 下。
3. **修正玩家存檔**:用社群工具
   [palworld-host-save-fix](https://github.com/xNul/palworld-host-save-fix)
   把主機玩家的 `.sav` 轉成專用伺服器格式。你需要準備:
   - 舊的 host 存檔檔名(通常是 `00000000000000000000000000000001.sav`)
   - 每位玩家新的 PlayerUid(讓他們先各自進伺服器一次建立空角色,
     GUI 的**存檔備份**分頁會列出新產生的玩家存檔檔名)
4. 回到 GUI,「設為啟用世界」→ 啟動。

> 沒有工具能百分之百保證主機角色轉移成功。**動手前先在 GUI 按「立即備份」**,
> 出事可以一鍵還原。

## 遷移後的檢查清單

- [ ] **存檔備份**分頁裡,你的世界標示為「啟用中」
- [ ] 世界大小合理(空世界通常不到 1 MB;玩過的世界幾十 MB 起跳)
- [ ] 玩家存檔數量符合預期
- [ ] 啟動後日誌沒有 `Save data is corrupted`
- [ ] 進遊戲確認建築、帕魯、等級都在

## 常見問題

**進去變成全新世界,舊的東西都不見了**
`DedicatedServerName` 沒指到你的 GUID。到**存檔備份**分頁按「設為啟用世界」。

**世界在,但要我重建角色**
情境 C 的 PlayerUid 問題,見上方第 3 步。若是從專用伺服器搬到專用伺服器,
檢查 `Players/` 資料夾是不是漏掉沒複製。

**啟動就 `Save data is corrupted` 然後崩潰**
存檔在複製時是開著的(來源端沒關乾淨),或傳輸過程截斷。用備份還原,
重新從來源端複製一次。

**跨平台(Windows 伺服器 ↔ Linux 伺服器)可以嗎?**
可以,`.sav` 格式相同。注意設定檔在 `Config/WindowsServer/` 與
`Config/LinuxServer/` 兩個不同資料夾,GUI 會依 agent 所在平台自動處理。

---

## 順帶一提:自動備份

遷移完成後,建議到**存檔備份**分頁把**自動備份**打開。agent 會依你設定的間隔,
在伺服器運作時定期打包啟用中的世界,超過保留份數的舊備份自動清除;
勾了「沒有玩家在線上時跳過」就不會堆積一堆一模一樣的檔案。

備份前 agent 會先請伺服器把世界寫入磁碟(透過 REST API),所以拿到的不是
寫到一半的快照 —— 前提是 `RESTAPIEnabled` 有開、`AdminPassword` 有設。
